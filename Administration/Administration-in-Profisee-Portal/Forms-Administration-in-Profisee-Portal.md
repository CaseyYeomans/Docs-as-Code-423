# Forms Administration in Profisee Portal

Profisee administrators can perform Forms Administration functions in Profisee Portal. Forms are an administrative feature designed to build and configure HTML templates. The Forms feature provides great flexibility for designing forms, making it possible for administrators to create custom forms, allowing workflow participants to interact with master data through Profisee Portal and workflows in countless ways.

Using the **Forms** page in the administration area of the portal, you can create, edit, import and export, and manage form configurations to be used elsewhere in the Profisee Portal.

![](https://Profisee.magentrixcloud.com/sys/staticasset/read/file-i6065649be340de24.png)

From here, a list of all created Forms is displayed with the following information:

* **Name**: The name given for the Form
* **Entity**: The entity the form is associated with
* **Default Form**: Indicates whether this form is set as the default for the entity
* **Updated On**: The datetime the form was last edited
* **Updated By**: The last user who edited the form
* **Created On**: The datetime the form was created
* **Created By**: The user who created this form

![](https://Profisee.magentrixcloud.com/sys/staticasset/read/file-i35283dfc57a1de7.png)

If desired, you can right-click a form (or select multiple) in this list to bring up the context menu, allowing you to **Delete** the selected forms. ![](https://Profisee.magentrixcloud.com/sys/staticasset/read/file-i88516df3a59cae02.png)

You can also click the ellipsis menu at the top of the grid to access additional options:

* **Import Form**: Opens a dialog to upload and import existing form configurations from a .maestroform file.
* **Export Form**: Exports the selected form configuration(s) into a JSON file compatible with the Forms admin API endpoints.

You can use the [Quick Filter](https://support.profisee.com/wikis/profiseeplatform/use_the_quick_filter) to search for specific items in the forms list.
